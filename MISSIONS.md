# CDMAS Frontend Missions

Nine improvements split into three cohesive tracks, one per group member. Each
track groups missions that touch related parts of the UI so the work stays
focused and merges cleanly.

---

## Ran — Track A: Live Flow & Animation

This track is about making the simulation feel alive: animating what moves
through the network and giving the user fine-grained control over time.

- [ ] **M1 — Animated message flow.** Make the agent message flow more
  interactive: show animation of packets traveling between agents.
- [ ] **M3 — Slow-motion mode.** Add a slow-motion button that lets us watch
  everything happening across the network step by step.
- [ ] **M4 — Inside-out visualization.** Enhance the visualization so we can see
  each part of the lifecycle — from an agent's formation all the way to its
  decision and onward.

---

## Adi — Track B: Inspector & Data Panels

This track is about the detail panels: making the tabs that expose per-agent
data clearer, more honest, and more user friendly.

- [ ] **M5 — Friendlier Agent Inspector.** Make the Agent Inspector tab more
  user friendly.
- [ ] **M6 — Transparent Validator.** Make the Validator tab actually show where
  and how each check was performed.
- [ ] **M8 — Resource allocation detail.** Improve the resource allocation view
  so we can see how much each agent consumed.

---

## Eden — Track C: Controls, Topology & Cleanup

This track is about navigation and presentation: how the user drives scenarios,
reads the network layout, and a polish pass on stray wording.

- [ ] **M2 — Scenario buttons.** Instead of a single play button, add a button
  for each scenario.
- [ ] **M7 — Clearer network topology.** Expand the network topology view so its
  meaning is obvious at a glance.
- [ ] **M9 — Remove "SRS" references.** Remove the "SRS" wording/references from
  the frontend.
